  <action>
Create `frontend/components/CommentSheet.tsx`. Follow AuthBottomSheet's layout/animation pattern exactly.

**Props interface:**
```typescript
interface CommentSheetProps {
  isOpen: boolean
  videoId: string | null
  currentUserId: string | null  // null = guest
  onClose: () => void
}
```

**Internal Comment type:**
```typescript
interface Comment {
  id: string
  body: string
  created_at: string
  user_id: string
  profiles: { display_name: string | null; avatar_url: string | null } | null
}
```

**State:**
- `comments: Comment[]` — loaded from GET /api/comments?videoId=...
- `loading: boolean` — while fetching
- `input: string` — current textarea value
- `submitting: boolean` — while POST in flight
- `submitError: string | null` — error from POST (rate limit, server error)

**Data loading:**
Use `useEffect` keyed on `[isOpen, videoId]`. When `isOpen && videoId`, fetch `/api/comments?videoId=${videoId}`. Set comments on success. Reset comments to [] when sheet closes (isOpen becomes false). Do NOT fetch comments on every video render — lazy load only when sheet opens.

**Layout (inline styles, matches AuthBottomSheet z-index hierarchy):**
- Overlay div: fixed inset-0, background rgba(0,0,0,0.6), zIndex 100, pointer-events none when closed, onClick calls onClose
- Panel div: fixed bottom-0 left-0 right-0, height: '75vh', background '#1a1a1a', borderRadius '20px 20px 0 0', zIndex 101
  - transform: translateY(0) when open, translateY(100%) when closed
  - transition: 'transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)'
  - paddingBottom: 'env(safe-area-inset-bottom)' for iOS
  - display flex, flexDirection column

**Panel content (top to bottom):**
1. Drag handle pill — onClick calls onClose; also closes on overlay click
2. Header: "Comments" text (left) + comment count `(${comments.length})` (right)
3. Scrollable comment list (flex: 1, overflowY: 'auto'):
   - `e.stopPropagation()` on the scroll container's onTouchMove/onWheel to prevent video feed scroll
   - Empty state: "No comments yet. Be the first!" centered
   - Each comment row:
     - Avatar circle (40px): first letter of display_name or '?' for null, background '#333', white text
     - display_name (bold) or 'Anonymous' fallback
     - Relative time: use `new Date(comment.created_at).toLocaleTimeString()` style (simple, no library)
     - Comment body text
     - Trash icon button: ONLY render when `comment.user_id === currentUserId`. On click: call DELETE /api/comments/${comment.id}, then remove comment from local state
4. Input area (fixed at bottom of panel):
   - If currentUserId is null (guest): show "Sign in to comment" text — no textarea
   - If signed in: textarea (maxLength={500}, placeholder "Add a comment...") + char counter `${input.length}/500` + Submit button
     - Submit button: disabled when input.trim().length === 0 || submitting
     - On submit: add optimistic comment to list (use a temp id), clear input, POST to /api/comments, replace optimistic comment with server response on success, or remove optimistic comment + set submitError on failure
     - If submitError is set, show it in red above the input
     - Rate limit error (429 response): display the server's error message ("Please wait 30 seconds between comments.")
