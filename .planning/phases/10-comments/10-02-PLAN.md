**GET /api/comments?videoId=uuid (public — no auth required):**
- Extract `videoId` from searchParams; return 400 if missing
- Use anon client (`createClient` from `@supabase/supabase-js` with NEXT_PUBLIC_SUPABASE_URL + NEXT_PUBLIC_SUPABASE_ANON_KEY) — NOT the SSR cookie client (GET is public, anon client is correct here; matches two-client pattern from Phase 9 /api/social/state)
- Select from video_comments with embedded profiles join:
  ```
  id, body, created_at, user_id,
  profiles ( display_name, avatar_url )
  ```
- Filter `.eq('video_id', videoId)`, order `.order('created_at', { ascending: true })`
- Return `{ comments: comments ?? [] }` on success, `{ error }` with 500 on error

**POST /api/comments (auth-gated):**
- Use SSR cookie client: `import { createClient } from '@/lib/supabase/server'`
- `getUser()` — return 401 if no user
- Parse request JSON: `{ videoId, body }`
- Validate: return 400 if videoId missing, body empty or whitespace-only, or body.length > 500
- Rate limit: query video_comments for `.eq('user_id', user.id).gte('created_at', thirtySecondsAgo).limit(1).maybeSingle()` where `thirtySecondsAgo = new Date(Date.now() - 30_000).toISOString()`. DO NOT add a video_id filter — rate limit is per-user cross-video (COMM-04 pitfall 4). Return 429 with `{ error: 'Please wait 30 seconds between comments.' }` if recent exists.
- Insert: `{ video_id: videoId, user_id: user.id, body: body.trim() }`
- Return the inserted row with profiles join (same select as GET) using `.select(...).single()`
- Return 201 with `{ comment }` on success

See 10-RESEARCH.md Pattern 2 for the exact implementation.

**DELETE /api/comments/[id] (auth-gated, owner-only):**
- Use SSR cookie client: `import { createClient } from '@/lib/supabase/server'`
- `getUser()` — return 401 if no user
- Await `params` (Next.js 15+ params is a Promise): `const { id } = await params`
- Delete: `.from('video_comments').delete().eq('id', id).eq('user_id', user.id).select('id').maybeSingle()`
  - The `.eq('user_id', user.id)` is defense-in-depth; RLS also enforces ownership
- If `data` is null after delete: return 404 `{ error: 'Not found or not authorized' }` — this covers both "doesn't exist" and "not owner" cases
- If error: return 500 `{ error: error.message }`
- On success: return 200 `{ deleted: true }`

Function signature:
```typescript
export async function DELETE(
  _req: Request,
  { params }: { params: Promise<{ id: string }> }
)
```
