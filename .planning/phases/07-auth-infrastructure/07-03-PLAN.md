<task type="checkpoint:human-action">
  <name>Task 2: Create Google OAuth credentials and configure Supabase Google provider</name>
  <files>n/a — external dashboard configuration, no files modified</files>
  <action>
Step A — Create OAuth 2.0 credentials in Google Cloud Console:
1. Go to https://console.cloud.google.com. Select or create a project (e.g. "FinFeed").
2. Go to APIs & Services → OAuth consent screen.
   - User type: External
   - App name: FinFeed
   - Add your email as a test user.
3. Go to APIs & Services → Credentials → Create Credentials → OAuth client ID.
   - Application type: Web application
   - Name: FinFeed Web
   - Authorized JavaScript origins: http://localhost:3000 and https://autonews-ai.vercel.app
   - Authorized redirect URIs: https://[YOUR-SUPABASE-PROJECT-ID].supabase.co/auth/v1/callback
     (Find project ID in Supabase Dashboard → Settings → General)
4. Copy the Client ID and Client Secret.

Step B — Enable Google provider in Supabase:
1. Go to Supabase Dashboard → Authentication → Providers → Google.
2. Toggle Google to enabled.
3. Paste Client ID and Client Secret from Step A.
4. Click Save.

Step C — Add allowed redirect URLs in Supabase:
1. Go to Supabase Dashboard → Authentication → URL Configuration.
2. Under "Redirect URLs" add all three:
   - http://localhost:3000/**
   - https://autonews-ai.vercel.app/**
   - https://*-autonews-ai.vercel.app/**
3. Set Site URL to: https://autonews-ai.vercel.app
4. Click Save.
  </action>
