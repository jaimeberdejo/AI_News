    **4. Add handleSocialAction function** (after the existing `handleReplay` and `handleNewEdition` functions):
    ```typescript
    function handleSocialAction(action: 'like' | 'bookmark' | 'comment') {
      if (authLoading) return  // debounce: auth state not yet known
      if (isGuest) {
        setSheetAction(action)
      }
      // signed-in: no-op for now (Phase 9 adds real handlers)
    }
    ```

    **5. Add scroll restoration useEffect** (add after the existing category-switching useEffect):
    ```typescript
    useEffect(() => {
      const idx = searchParams.get('videoIndex')
      if (idx !== null && feedRef.current && videos.length > 0) {
        const target = parseInt(idx, 10)
        if (!isNaN(target) && target >= 0 && target < videos.length) {
          feedRef.current.scrollTop = target * feedRef.current.clientHeight
          setActiveIndex(target)
        }
        router.replace('/', { scroll: false })
      }
    }, [videos.length]) // eslint-disable-line react-hooks/exhaustive-deps
    // Intentional: only run once after videos load, not on every searchParams change.
    // router and searchParams are stable references — adding them causes double-fire.
    ```

    **6. Update VideoItem props** — add `onSocialAction` to every VideoItem rendered in the `videos.map(...)` block:
    ```typescript
    onSocialAction={handleSocialAction}
    ```

    **7. Render AuthBottomSheet** — add it as the last child inside the outer container `<div>` (after the scroll container `<div ref={feedRef}>`):
    ```tsx
    <AuthBottomSheet
      isOpen={sheetAction !== null}
      actionLabel={
        sheetAction === 'like' ? 'like this' :
        sheetAction === 'bookmark' ? 'bookmark this' :
        'comment'
      }
      returnPath={`/?videoIndex=${activeIndex}`}
      onClose={() => setSheetAction(null)}
    />
    ```

    **Architecture notes:**
    - AuthBottomSheet is rendered once at the VideoFeed level — NOT inside VideoItem. One instance, controlled by VideoFeed state.
    - The `authLoading` debounce in `handleSocialAction` prevents a signed-in user from briefly seeing the sheet before `getUser()` resolves on mount.
    - The `videos.length` dep in scroll restoration is intentional — waits until videos array is populated before attempting scroll.
    - `router.replace('/', { scroll: false })` must be called regardless of whether target index was valid, to clean up the URL.

    }
