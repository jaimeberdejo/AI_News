  <action>
Create the migration file at `frontend/supabase/migrations/20260325000001_add_social_tables.sql` with the following contents (use IF NOT EXISTS guards throughout):

1. Add `like_count integer NOT NULL DEFAULT 0` to `public.videos` using `ALTER TABLE public.videos ADD COLUMN IF NOT EXISTS like_count integer NOT NULL DEFAULT 0`.

2. Create `public.video_likes` table:
   - `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
   - `video_id uuid NOT NULL REFERENCES public.videos(id) ON DELETE CASCADE`
   - `created_at timestamptz DEFAULT now()`
   - `UNIQUE (user_id, video_id)`
   - Two indexes: `idx_video_likes_video ON public.video_likes(video_id)` and `idx_video_likes_user ON public.video_likes(user_id)`
   - `ALTER TABLE public.video_likes ENABLE ROW LEVEL SECURITY`
   - RLS policy "anyone can read likes": `FOR SELECT TO authenticated, anon USING (true)` — SOCL-02 requires guests to see counts
   - RLS policy "users can insert own likes": `FOR INSERT TO authenticated WITH CHECK (auth.uid() = user_id)`
   - RLS policy "users can delete own likes": `FOR DELETE TO authenticated USING (auth.uid() = user_id)`

3. Create `public.video_bookmarks` table:
   - Same shape as video_likes but bookmarks are PRIVATE
   - `id, user_id, video_id, created_at, UNIQUE (user_id, video_id)`
   - Index: `idx_video_bookmarks_user ON public.video_bookmarks(user_id)`
   - `ALTER TABLE public.video_bookmarks ENABLE ROW LEVEL SECURITY`
   - RLS policy "users can read own bookmarks": `FOR SELECT TO authenticated USING (auth.uid() = user_id)` — no anon SELECT
   - RLS policy "users can insert own bookmarks": `FOR INSERT TO authenticated WITH CHECK (auth.uid() = user_id)`
   - RLS policy "users can delete own bookmarks": `FOR DELETE TO authenticated USING (auth.uid() = user_id)`

4. Trigger function `public.update_video_like_count()`:
   - `RETURNS trigger LANGUAGE plpgsql SECURITY DEFINER SET search_path = ''`
   - On INSERT: `UPDATE public.videos SET like_count = like_count + 1 WHERE id = NEW.video_id`
   - On DELETE: `UPDATE public.videos SET like_count = GREATEST(like_count - 1, 0) WHERE id = OLD.video_id`
   - `RETURN NULL`
   - Use `CREATE OR REPLACE FUNCTION`

5. Trigger `on_like_change`:
   - `AFTER INSERT OR DELETE ON public.video_likes FOR EACH ROW EXECUTE FUNCTION public.update_video_like_count()`
   - Use `CREATE TRIGGER` (not CREATE OR REPLACE — triggers don't support that in older PG; wrap with `DROP TRIGGER IF EXISTS on_like_change ON public.video_likes` before CREATE TRIGGER)

Follow the exact pattern from `frontend/supabase/migrations/20260323000001_add_profiles_and_auth.sql` (SECURITY DEFINER + SET search_path = '' on the trigger function — same CVE protection).
