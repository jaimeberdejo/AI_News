# Milestone v1.3: Demo Prep

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 13
**Total Plans:** 2

## Overview

Focused demo-prep milestone. Replaced the Google-OAuth-only registration path with a frictionless email + password form so user-testing demo participants can register without a Google account and without email confirmation friction. All social features remain gated for signed-in users; this milestone makes it trivially easy for demo guests to become signed-in users.

## Phases

### Phase 13: Demo Auth Simplification

**Goal:** Replace Google OAuth registration with a simple email + password + confirm-password form so demo users can register without friction, landing immediately in the app with email confirmation disabled
**Depends on:** Phase 12
**Plans:** 2/2 plans complete

Plans:

- [x] 13-01: Tabbed login page (Sign In + Register) + signUp action redirects to '/'
- [x] 13-02: AuthBottomSheet "Create account" link + Supabase email confirmation disabled

**Details:**
- `/auth/login` converted from single Google button to two-tab UI; Register tab is the default
- `signUp` Server Action now calls `redirect('/')` on success — no check-your-email gate
- `AuthBottomSheet` gains a plain `<a href="/auth/login">Create account</a>` anchor below the Google button
- Supabase "Confirm email" toggle disabled project-wide (Auth → Providers → Email)
- Google OAuth path in AuthBottomSheet and login page completely preserved; iOS PWA fix untouched

## Milestone Summary

**Key Decisions:**

- `signUp` calls `redirect('/')` not `return { message }` — demo users land in app immediately; re-enabling email confirmation requires reverting this
- Default `/auth/login` tab is `register` (not `signin`) — optimises for demo UX where new users register first
- Plain `<a href>` anchor in AuthBottomSheet (not next/link) — consistent with existing `window.location.href` pattern in the same component; full-page navigation preferred for auth routes
- Supabase "Confirm email" disabled at project level — noted in PROJECT.md as a demo trade-off; must be re-evaluated before scaling to real users

**Issues Resolved:**

- Demo registration friction: new users previously required a Google account or a working email confirm flow

**Issues Deferred:**

- Re-enabling email confirmation for production (requires signUp action change + Supabase toggle)
- Apple Sign In — still deferred to v1.4+

**Technical Debt Incurred:**

- `signUp redirect('/')` + Supabase "Confirm email" OFF is a demo-only configuration — must revert before targeting real (non-demo) users at scale
