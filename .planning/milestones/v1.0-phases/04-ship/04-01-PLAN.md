  <action>
Create `.github/workflows/pipeline.yml` with exactly this structure:

```yaml
name: FinFeed Pipeline

on:
  schedule:
    - cron: '0 6 * * *'    # 6am UTC daily
    - cron: '0 18 * * *'   # 6pm UTC daily
  workflow_dispatch:         # Manual trigger for testing

jobs:
  run-pipeline:
    runs-on: ubuntu-latest
    timeout-minutes: 45

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Cache pip dependencies
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: ${{ runner.os }}-pip-${{ hashFiles('requirements.txt') }}
          restore-keys: |
            ${{ runner.os }}-pip-

      - name: Cache HuggingFace model (faster-whisper tiny.en)
        uses: actions/cache@v4
        with:
          path: ~/.cache/huggingface
          key: huggingface-faster-whisper-tiny-en-v1

      - name: Install system dependencies
        run: |
          sudo apt-get update -qq
          sudo apt-get install -y ffmpeg

      - name: Install Python dependencies
        run: pip install -r requirements.txt

      - name: Run pipeline
        env:
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_KEY: ${{ secrets.SUPABASE_SERVICE_KEY }}
          GROQ_API_KEY: ${{ secrets.GROQ_API_KEY }}
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          PEXELS_API_KEY: ${{ secrets.PEXELS_API_KEY }}
        run: python -m pipeline.run
```

Key decisions to enforce:
- `timeout-minutes: 45` not 30 — research open question #1: first cold-cache run may be 20-28 min; 45 gives headroom. Tighten to 30 after measuring first runs.
- `workflow_dispatch` is REQUIRED alongside schedule — without it, you cannot manually trigger to test before waiting for cron window.
- Secrets in `env:` block on the Run Pipeline step ONLY — NOT at job or workflow level, NOT as CLI args (visible in ps output).
- `apt-get update -qq` (quiet mode) — suppresses verbose mirror output that clutters logs.
- `actions/cache@v4` NOT v3 — v3 deprecated Feb 2025, v4 required for new cache backend service.
- HuggingFace cache key is static (`huggingface-faster-whisper-tiny-en-v1`) — model weights are deterministic and don't change between runs.
- pip cache key uses `hashFiles('requirements.txt')` with restore-keys fallback — cache miss with no fallback means every requirements.txt change starts from zero.
- Python version pinned to `'3.11'` — matches the local development environment.
- `runs-on: ubuntu-latest` — has apt, Python, and is the standard CI runner.

DO NOT:
- Add `secrets: inherit` or pass secrets at the job level
- Print any env var with echo or in any run: command
- Use `actions/cache@v3` (deprecated)
- Skip `workflow_dispatch` — it is required for manual testing before cron fires
- Schedule at midnight UTC (00:00) — GitHub Actions docs warn this time has maximum queue delay
