<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Deploy to Vercel via dashboard and configure environment variables</name>
  <files></files>
  <action>
Deploy the frontend to Vercel and configure environment variables. This step requires dashboard interaction — Vercel's GitHub integration cannot be set up via CLI without pre-existing auth tokens.

**Step 1: Import the repository**
1. Go to https://vercel.com/new
2. Click "Import Git Repository"
3. Select the `AutoNews_AI` repository
4. Set **Root Directory** to `frontend` (critical — without this, Vercel builds from repo root and fails)
5. Framework should auto-detect as "Next.js"
6. Click "Deploy" (this first deploy will likely fail or show no data — that's expected before env vars are set)

**Step 2: Note the deployment URL**
After the first deploy completes, note your project URL. It will be something like:
`https://auto-news-ai-[hash].vercel.app`

**Step 3: Add environment variables**
Go to Project Settings → Environment Variables. Add these three variables for the **Production** environment:

| Name | Value |
|------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://yfryhktlkbemzyequgds.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | *(your anon key from Supabase Dashboard → Settings → API → Project API Keys → anon/public)* |
| `NEXT_PUBLIC_APP_URL` | `https://your-project.vercel.app` *(use the URL from Step 2)* |

**CRITICAL:** `NEXT_PUBLIC_APP_URL` must be the production Vercel URL, NOT `http://localhost:3000`. The server component in `page.tsx` fetches `/api/today` using this URL during SSR on Vercel's servers. If it points to localhost, the page will render with no videos.

**Step 4: Redeploy**
After adding the env vars, go to Deployments → click the three dots on the latest deployment → "Redeploy". This will build with the correct environment variables.

**Step 5: Verify**
After redeployment, open the production URL in a browser. You should see:
- The FinFeed app loads
- Either videos play (if a pipeline run has produced an edition) OR the app shows a graceful empty state
- No red errors in the browser console related to CORS or missing environment variables
