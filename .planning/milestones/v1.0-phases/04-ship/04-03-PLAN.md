**Step 1: Add secrets via GitHub CLI**
Run these commands from the repo root, reading values from the existing `.env` file:

```bash
cd /Users/jaimeberdejosanchez/projects/FinFeed

gh secret set SUPABASE_SERVICE_KEY --body "$(grep '^SUPABASE_SERVICE_KEY=' .env | cut -d'=' -f2-)"

```

If a `grep` command returns empty (the key is missing from `.env`), the secret will be set to an empty string — the workflow will fail with an auth error. Check `.env` and set the value manually if needed:

```bash

```

This run has no warm caches — expect 15-25 minutes. The `gh run watch` command streams live output. Watch for:
- `Set up Python` step completing (fast)
- `Cache pip dependencies` — first run will be a cache miss, that is OK
- `Cache HuggingFace model` — first run will be a cache miss, that is OK
- `Install system dependencies` — apt-get ffmpeg, expect 1-2 minutes
- `Install Python dependencies` — pip install, expect 2-3 minutes first run
- `Run pipeline` — the actual pipeline, expect 5-15 minutes depending on news day

If the run times out at 45 minutes: run `gh run view --log` to check where it stalled.
